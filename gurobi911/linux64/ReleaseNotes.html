Gurobi Optimizer 9.1 uses the same licenses as version 9.0, so if you already have a Gurobi 9 license, there's no need to obtain a new one. If you don't already have a Gurobi 9 license, you will need to visit the Gurobi License Center. If you are a commercial user under maintenance, you should see your Gurobi 9 license under Current Gurobi Licenses. If you would like to request a free academic license, you can do so from the Free Academic License page. Once you have a license on the Gurobi web site, you will need to follow the instructions for installing a license in the Quick Start Guide.

Release Notes for Version 9.1

In addition to performance enhancements, Gurobi 9.1 adds the following new features:

  • A New Heuristic For Models with Expensive Relaxations: Our new NoRel heuristic finds high-quality solutions in situations where the linear programming (LP) relaxation of a mixed-integer programming (MIP) problem is very expensive to solve. It is controlled by parameters NoRelHeurTime and NoRelHeurWork.
  • Integrality Focus: This new feature forces the solver to be much stricter about integrality constraints, thus avoiding many undesirable results (including trickle flows) that can come from small integrality violations. It is controlled by the IntegralityFocus parameter.
  • Further Python Matrix API Enhancements: Gurobi's Python interface - gurobipy - has been extended and improved to better support matrix-oriented modeling. Consult the documention for the new MConstr object for details.
  • Pip Install Support: Users can now utilize pip, the package installer for Python, to install gurobipy into their Python environment.
  • Releasing the GIL in Python API: When the optimize() method is called, gurobipy now releases the Global Interpreter Lock (GIL), which allows user programs to execute Python code in another Python thread while optimize() is running.
  • Tuning Tool Enhancements: We added a number of additional controls to our tuning tool. See the Parameter Tuning Tool documentation for more details.
  • Record/Replay for Compute Server and Cloud: We now support the record/replay feature for Gurobi Compute Server and Gurobi Instant Cloud. See Recording and Replay for more details.
  • Pre-specified User Cuts: By setting the Lazy linear constraint attribute to the new value -1, the user can declare a linear constraint to be a user cut. The constraint must be redundant with respect to the rest of the model. The solver can optionally add user cuts to the relaxation in order to cut off LP solutions that are encountered during the MIP solving process and potentially improve performance.
  • New Parameters and Attributes: In addition to the attributes and parameters directly tied to the new features above, there is one additional new parameter and one additional new attribute.
    • PoolGapAbs New parameter to specify the maximum absolute gap allowed for stored solutions.
    • FingerPrint New attribute to query the model fingerprint.

Code Conversion from Version 9.0

Programs that were originally written for version 9.0 of the Gurobi Optimizer generally won't require any code changes to use version 9.1. If you'd like to use the new features, some code changes are required.
